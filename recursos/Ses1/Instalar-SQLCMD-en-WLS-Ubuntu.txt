Previo a la instalación en SSMS: (8)

1.  Conectar al servidor local (autenticación windows)
2.  Seguridad -> Inicios de sesión -> sa
3.  Click derecho en sa -> propiedades -> configurar pass: 1234
4.  Marcar únicamente: Exigir directivas de contraseña -> Aceptar

Instalar-SQLCMD-en-WSL-Ubuntu: (15)

1.  sudo apt-get update
2.  sudo apt-get install curl gnupg -y
3.  curl https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -
4.  curl https://packages.microsoft.com/config/ubuntu/22.04/prod.list | sudo tee /etc/apt/sources.list.d/msprod.list
5.  sudo apt-get update
6.  sudo apt-get install mssql-tools unixodbc-dev
7.  echo 'export PATH="$PATH:/opt/mssql-tools/bin"' >> ~/.bashrc
8.  source ~/.bashrc
9.  which sqlcmd (Comprobar la instalación) -> debe verse: /opt/mssql-tools/bin/sqlcmd

Conectando a SQL Server desde SQLCMD en WSL: (2)

1.  ip route show -> ejemplo: default via 172.30.16.1
2.  sqlcmd -S 172.30.16.1 -U sa -P "TU_PASSWORD"

Sino funciona: (15)

1.  win + r
2.  SQLServerManager16.msc (Para SQL Server 2022 únicamente)
3.  SQL Server Network Configuration
4.  Protocols for MSSQLSERVER
5.  TCP/IP -> Enable (para poder conectar con WSL)
6.  Sobre TCP/IP click derecho -> Propiedades -> pestaña IP Addresses
7.  En IPALL verificar en TCP Port el puerto 1433 (si no está, ajustarlo)
8.  En PowerShell: Crear una regla de firewall en Windows (para permitir conexiones entrantes a SQL Server)
     -> New-NetFirewallRule -DisplayName "SQLServer1433" -Direction Inbound -Protocol TCP -LocalPort 1433 -Action Allow
9.  (Opcional) Si después queremos borrarla: Remove-NetFirewallRule -DisplayName "SQLServer1433"
